Role & Project Details

Role: UX Researcher & Designer (Solo Project)

Timeline: May 2025

Focus: Accessibility, Emotional UX, Voice User Interfaces (VUI), Cross-Cultural Design

Project Overview

This case study explores how UX design can improve smart home interfaces for older adults. With the global aging population, there‚Äôs a growing need for interfaces that prioritize trust, emotional engagement, and accessibility. This project focused on designing a tablet-based smart home experience that supports independence, cultural relevance, and intuitive use for users aged 60 and above.

The Problem

  • Small touch targets and cluttered interfaces
  • Limited voice recognition accuracy for accents, slower speech, or non-standard phrasing
  • Cold or impersonal user interactions that discourage trust
  • Complex navigation paths and poor visual hierarchy
  • Lack of emotional support or feedback mechanisms

These limitations can cause frustration, technophobia, and abandonment of smart home systems ‚Äî especially in older adults managing cognitive or physical changes.

Design Question: How might we create emotionally supportive, intuitive smart home interfaces that promote trust, autonomy, and accessibility for elderly users?

Research Approach

Academic Research

  • Voice-user interface (VUI) usability among elderly users
  • Emotional design in smart environments
  • Cross-cultural UX, with a focus on Chinese elderly populations
  • Trust and transparency in automation
  • Aging, memory, and cognition in interface design

In-Person Testing

Conducted moderated usability testing with three participants (all over 60) using a tablet prototype. Observed task performance, noted pain points, and gathered emotional feedback.

Key Insights

  1. Accessibility requires more than large buttons. Simplified layouts and cognitive support features like checklists improved usability.
  2. Emotional design builds trust. Warm visuals and encouraging feedback increased user comfort.
  3. Simplicity reduces friction. Clear, linear flows improved task completion.
  4. Personalization matters. Customizable voice tone, font size, and reminders empowered users.
  5. VUI success depends on natural speech. Short, forgiving commands improved recognition accuracy.

Design Decisions

  • Voice-activated breathing orb for calming, humanized interaction
  • Large, high-contrast buttons and text for readability
  • Clear, linear task flow with minimal branching
  • Encouraging visuals and soft sounds for emotional reinforcement
  • Simple, natural-language voice commands
  • Tablet-first design for accessibility and stability

Final Solution

The final prototype centers on a glowing, responsive orb paired with accessible UI elements. It supports natural voice commands, shows friendly confirmations, and keeps navigation linear and intuitive.

  • Welcome screen with time, weather, and breathing orb
  • Voice command interface with readable feedback
  • Reminders with icons and friendly text
  • Control panels for lights, medication, and check-ins

Reflection & Outcome

This project deepened my understanding of accessibility and emotional design. Observing participants connect with the orb confirmed that design can make technology feel approachable, not intimidating.

Next Steps

  • Expand user testing across diverse backgrounds
  • Prototype mobile and smartwatch versions
  • Refine voice interface for tone and timing
  • Collaborate with developers for real-world implementation
  • Conduct long-term adoption studies
