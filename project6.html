Infinite Coloring Book - UX Case Study

by Breanna Gushman

Project Overview

Role: UX Researcher & Designer
Timeline: Fall 2025 (3 Milestones)
Team: 4-person interdisciplinary team
Tools: Balsamiq, Figma, Survey Design, Usability Testing, Heuristic Evaluation

The Challenge

Current digital coloring apps feel cluttered, lack personalization, and don't allow users to turn their own photos into coloring pages. Users seek calming, accessible creative outlets but are frustrated by paywalls, intrusive ads, and confusing interfaces.

The Solution

Infinite Coloring Book transforms user photos into customizable color-by-number or free-coloring templates with multiple art styles (Pop Art, Watercolor, Comic), creating a calming, personalized digital coloring experience with seamless sharing capabilities.

My Role & Contributions

Milestone 1: Research & Discovery

  • Survey Design & Distribution: Created and distributed comprehensive user survey to gather feedback on user experiences, accessibility needs, and design preferences for coloring apps
  • Data Analysis: Analyzed survey responses, identified common trends and key insights, and summarized findings to support persona creation and functionality development
  • Key Insight Discovery: Uncovered that users prioritize wide color palettes (4.18/5), no ads/paywalls (3.86/5), and calming interfaces (3.5/5)

Milestone 2: Low-Fidelity Prototyping

  • Balsamiq Prototyping: Created multiple critical screens including:
    • Home page navigation structure
    • Share with friends flow
    • Interactive coloring page layout
    • Photo capture interface
  • A/B Testing Design: Designed A/B prototype variations for camera navigation to center user needs and persona requirements (Type A: traditional back button vs. Type B: gesture-based swipe navigation inspired by Snapchat)

Milestone 3: Evaluation & High-Fidelity Design

  • Note-Taking & Observation: Documented usability testing session, capturing real-time user confusion around hexagon button and interface elements
  • High-Fidelity Prototype: Created complete high-fidelity prototype translating all lo-fi insights into polished, functional screens
  • Design Documentation: Wrote comprehensive high-fidelity prototype documentation including problem statement evolution, feature differentiation from earlier milestones, and design rationale

Research Process

Literature Review Findings

Through competitive analysis of 3 leading apps (Lake, Pigment, UNICORN), I identified critical user pain points:

What Users Love:

  • No ads or intrusive interruptions
  • Calming interfaces with ASMR audio
  • Variety in brushes and color options
  • How-to videos and tutorials
  • Positive community features

What Frustrates Users:

  • Payment walls locking features
  • Unintuitive tool switching (finger vs. Apple Pencil for erasing)
  • Crashes causing lost progress
  • Limited customization options
  • Poor custom image-to-template conversion

User Survey Results

Distributed survey to diverse audience (family, friends, sports teams) yielding actionable insights:

Key Metrics:

  • Device Usage: Strong trend toward phone, laptop, and tablet use
  • Coloring Frequency: Mode response: 10-30 minute sessions

Feature Priorities:

  • Wide color palette: 4.18/5
  • No ads/paywalls: 3.86/5
  • Calming interface: 3.5/5
  • Accessibility options: 3.05/5

Critical Discovery: 50% of respondents don't currently use coloring apps, presenting massive opportunity for user acquisition through better design

Photo Upload Feature: Majority expressed interest in uploading own photos, validating core differentiator

Usability Testing & Insights

Testing Protocol

Conducted think-aloud usability tests with 3 users completing all 3 tasks while I observed and documented behaviors.

Usability Testing Setup

My Observation Notes

  • Hexagon confusion: Users unclear about hexagonal color picker button purpose
  • Scribbling enjoyment: Users loved creative/interactive elements
  • Art style label confusion: Users didn't connect "Art Style" buttons to color palette changes

Quantitative Results

  • Task 1 (Photo to Template): 11-13 sec, Avg 12 sec - Highly consistent
  • Task 2 (Change Palette): 12-34 sec, Avg 21.67 sec - High variance indicates confusion
  • Task 3 (Share): 8-48 sec, Avg 23.67 sec - High variance

Key Insight:

Task 1's consistency validated strong iconography. Tasks 2 & 3's variance revealed specific pain points requiring redesign.

Qualitative Findings

  • "The color palette dots are not colored, so it is hard to tell what they are"
  • "The weird stop sign button needs a defined purpose. Art style palette change was confusing"
  • "The control flow was intuitive and icons made sense"
  • "Drawing was fun, I like to scribble"
  • "Coloring was good"

User Action Analysis

  • Task 2 confusion: User 3 clicked color circles at bottom instead of "Art Style" buttons at top (34 sec vs. 12 sec for others)
  • Hexagon button: Ambiguous icon caused exploratory clicking during palette changes
  • Back button: Users uncertain where it would take them, causing hesitation
  • Continuity error: Blank save screen made User 1 question if template actually saved

Heuristic Evaluation

Conducted Nielsen Norman Group heuristic evaluation across all 10 principles for each task. Key findings I prioritized:

Critical Issues Identified

Visibility of System Status:

  • No color in lo-fi made palette selection invisible
  • No confirmation for successful photo capture or sharing
  • Solution: Add color indicators, confirmation popups

Match Between System and Real World:

  • Color palette selector at top, actual colors at bottom (user expectation violated)
  • Combined save/share button unlike standard conventions
  • Solution: Move palette controls near colors, separate save (floppy disk) and share (arrow) icons

User Control and Freedom:

  • No photo confirmation screen before template generation
  • Selected palette not indicated visually
  • Can't undo sharing or unfriend users
  • Solution: Add confirm/retake screen, selection indicators, undo options

Recognition Rather Than Recall:

  • Unlabeled tool buttons rely on icon recognition
  • Back buttons don't indicate destination
  • Solution: Add tooltips on hover/hold, contextual back button labels (home icon for gallery screen)

Affinity Diagram Insights

Task 1 (Photo to Template):

  • Lack of camera feedback/mock footage
  • Missing photo confirmation screen

Impact: Wasted processing on accidental photos

Task 2 (Change Palette):

  • Black & white lo-fi obscured color functionality
  • Confusing palette selector placement and labeling

Impact: Users couldn't identify color-related controls

Task 3 (Share):

  • Same-sized buttons (Title/Save/Share) caused confusion about order
  • Non-standard button shapes (text instead of icons)
  • Unclear back button destinations

Impact: Inconsistent completion times (8-48 sec)

Design Solutions & High-Fidelity Prototype

From Lo-Fi to Hi-Fi: My Design Evolution

What I Carried Forward:

  • Bottom navigation structure
  • Page flow and hierarchy
  • Core feature set

Critical Changes I Implemented:

Final UI Update 1

Final UI Update 2

Color Palette System Redesign

  • Moved palette selector into collapsible menu
  • Added visual color indicators on all buttons
  • Placed controls near actual color picker for spatial consistency
  • Result: Reduced cognitive load, matched mental models

Confirmation Screens

  • Added photo confirm/retake screen before template generation
  • Added "Sent successfully" popup for sharing
  • Result: Prevented wasted processing, provided system status feedback

Icon Improvements

  • Changed hexagon to universally recognized color picker icon
  • Separated save (floppy disk) and share (arrow) into distinct buttons
  • Added contextual back button labels
  • Result: Leveraged learned conventions, reduced errors

Visual Hierarchy

  • Differentiated button sizes/shapes for different functions
  • Minimized interface clutter with collapsible menus
  • Maximized coloring canvas space
  • Result: Focus on relaxation goal, reduced overwhelm

Accessibility Features

  • Maintained color-blind friendly numbering system
  • Added tool tooltips for clarity
  • Ensured multi-device responsiveness
  • Result: Inclusive design for broader user base

Impact & Results

  • Task 1 consistency (12 sec avg) validated intuitive photo-to-template flow worth preserving
  • Task 2 & 3 variance reduction through targeted redesigns addressing specific confusion points
  • User satisfaction improved through addressing all "confusing" survey feedback in hi-fi

Key Learnings

  • User-Centered Design Works: My survey data directly shaped feature priorities (color palette > realistic brushes > tutorials)
  • Lo-Fi Testing is Invaluable: Black & white prototype revealed color-dependent confusion that would've persisted to development
  • Observation > Assumptions: Video review uncovered specific interaction patterns (clicking wrong areas) that completion times alone wouldn't show
  • Iterative Design Process: Each milestone built on previous insights

Meeting User Needs

For Jane (Stressed Professional):

  • Quick 10-30 minute sessions supported
  • Auto-save prevents lost progress
  • Calming interface with soft colors
  • No paywalls or ads
  • Easy resume from home screen gallery

For Barty (Young Color-Blind User):

  • Color-by-number provides redundant labeling
  • Multi-device support (laptop-friendly)
  • Simplified, intuitive controls
  • Photo upload feature accessible

Reflection & Next Steps

What I'd Do Differently

  • Earlier color testing: Would create color lo-fi sooner to catch palette issues earlier
  • More diverse testers: 3 users revealed major issues, but broader demographics (age, ability) would provide richer insights
  • Competitive feature testing: Would conduct structured comparison tests with existing apps using same tasks

Future Enhancements

  • Collaborative coloring: Real-time shared template coloring with friends
  • Progress timelapse: Automatically capture drawing process for sharing
  • Community gallery: Upvote system for discovering popular templates
  • Advanced customization: Custom brush creation, texture options
  • Tutorial system: Guided first-time experience based on user feedback need (2.55/5 priority)
