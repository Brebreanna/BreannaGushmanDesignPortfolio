Infinite Coloring Book – Mobile App Design (In Progress)

A mobile coloring app that transforms user photos into customizable color-by-number templates, designed to support accessible, stress-reducing creative time for adults. Currently in the usability evaluation phase, with high-fidelity prototype development underway.

Role & Context

Role: UX Researcher & Designer
Course: CS 325 – Human-Computer Interaction
Duration: Fall 2025
Type: Team project (in progress)

I am responsible for leading research, synthesizing findings into personas and flows, and designing low-fidelity and early high-fidelity prototypes that can be evaluated and iterated on.

Problem & Goal

Many adults want low-effort, digital creative outlets that help them de-stress without feeling like they’re “bad at art.” Current coloring apps often feel childish, cluttered with ads, or inaccessible to users with different visual needs.

Goal: Design a mobile coloring experience that:

  • Transforms user photos into clear, color-by-number templates
  • Supports 10–30 minute “mini sessions” for stress relief
  • Is accessible for color-vision deficient users through redundant labeling strategies
  • Uses calming visuals and color choices grounded in psychological research

User Research & Analysis

I began with a mix of literature review, competitive analysis, and user surveying to understand both technical feasibility and user needs.

  • Conducted a literature review on edge detection approaches for photo-to-template conversion, including attention-based methods and color edge detection algorithms.
  • Reviewed psychological research on art therapy and color psychology to guide calming layout, typography, and color usage.
  • Investigated accessibility requirements for color-vision deficient users and planned redundant encoding (numbers + labels, not just color).
  • Performed a competitive analysis of three coloring apps (Lake, Pigment, UNICORN), identifying UX gaps such as unintuitive tool switching, limited customization, and paywalls that interrupt flow.

Survey Insights & Data-Driven Design

To move beyond assumptions, I designed and distributed a user survey capturing both quantitative metrics and qualitative feedback on habits, preferences, and accessibility.

  • Measured interest in features such as broad color palettes, ad-free experiences, and calming UI patterns.
  • Prioritized features based on ratings, including:
    • Wide color palettes: 4.18 / 5
    • Ad-free experience: 3.86 / 5
    • Calming interface elements: 3.5 / 5
  • Identified a target session length of 10–30 minutes, which informed quick-access patterns and the ability to easily resume in-progress pieces.

Persona Development

Using survey and research insights, I created personas in UXpressia that represent both the majority use case and key accessibility needs:

  • A primary persona: a 30-year-old professional seeking a calming, low-effort creative outlet after work.
  • An accessibility-focused persona: a 10-year-old color-blind user who still wants to fully participate in digital coloring activities.

These personas help the team balance “typical” user expectations with edge-case requirements instead of treating accessibility as an afterthought.

Prototyping & Iteration

I intentionally started with low-fidelity representations to validate flows and task structure before investing in high-fidelity visuals.

  • Sketched paper prototypes to quickly explore three core tasks: photo-to-template conversion, color palette customization, and social sharing.
  • Built Balsamiq wireframes mapping the user journey across 11+ screens.
  • Designed an A/B navigation prototype comparing traditional back-button navigation with gesture-based patterns, with a focus on keeping interactions simple.
  • Simplified Balsamiq designs into hand-drawn testable prototypes to reveal gaps between steps in multi-screen flows.

Usability Evaluation (Current Phase)

I designed a mixed-methods evaluation plan combining moderated usability testing, think-aloud protocol, and Nielsen’s 10 Usability Heuristics.

  • Tested the low-fidelity prototype with 3 participants, observing how they navigated core tasks and where confusion occurred.
  • Collected qualitative feedback on task completion, confusion points, and overall enjoyment.
  • Logged issues such as the need for clearer color palette indicators and more explicit button purposes.
  • Established an evaluation framework tracking task success rates, time-on-task, user frustration (Likert scale), observed issues, and heuristic violations.

Usability testing setup and observation using Balsamiq Low-fidelity Prototype.

Technical Approach

While this course project centers on UX, I also explored how a future implementation could handle photo-to-template conversion in a robust way.

  • Researched edge detection pipelines using attention-based approaches and RGB/HSV color channel integration to generate clean, color-by-number outlines.
  • Investigated how color edge detection could help differentiate fine details without overwhelming the final template.
  • Explored how LLM-based style transfer could eventually support artistic customization (e.g., watercolor or comic book styles) as a future enhancement.

Key Learnings & Next Steps

This project reinforced the importance of balancing majority needs with accessibility, and of testing low-fidelity designs before committing to polished visuals.

  • Accessibility isn’t a “nice-to-have”: Designing for color-vision deficiencies from the start changed how I thought about palettes, labels, and contrast.
  • Data over assumptions: Survey insights directly shaped feature priorities and the app’s session-length expectations.
  • Low-fi first: Paper sketches and Balsamiq wires helped catch flow gaps long before UI polish.

Next steps:

  • Complete the high-fidelity interactive prototype using findings from low-fi testing.
  • Run a final heuristic evaluation with Nielsen’s 10 Usability Heuristics.
  • Refine navigation clarity, feedback visibility, and accessibility features based on test results.
